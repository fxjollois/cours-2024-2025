$k$-plus proches voisins (ou $k$-NN - $k$-nearest neightbors)¶

L'ide de cette mthode, comme son nom l'indique, de rechercher un certain nombre de points proches de celui prdire. Ce nombre $k$ est un des paramtre de la mthode, l'autre tant la mthode utiliser pour calculer la distance entre deux points (trs souvent la distance euclidienne).

Cette mthode a l'avantage d'tre rapide, applicable des donnes de grandes tailles et ne ncessite pas d'hypothse forte. Par contre, si la mthode trouvent deux voisins une distance identique, selon le cas, elle peut n'en retenir qu'un des deux (le premier dans la liste). Et si la valeur de $y$ est diffrente entre les deux, cela impacte trs fortement les rsultats.

from sklearn.metrics import roc_curve, auc, precision_recall_curve

r2 = []
roc_auc = []
vn = []
fn = []
fp = []
vp = []

k_val = (1, 2, 5, 10, 15, 20)
for k in k_val:
    m_knn = KNeighborsClassifier(n_neighbors = k).fit(X, y)
    r2 = r2 + [m_knn.score(X, y)]
    fpr, tpr, th = roc_curve(y, pandas.DataFrame(m_knn.predict_proba(X))[1])
    roc_auc = roc_auc + [auc(fpr, tpr)]
    p = m_knn.predict(X) * 1
    vn = vn + [sum((1 - y) * (1 - p))]
    fn = fn + [sum((1 - y) * p)]
    fp = fp + [sum(y * (1 - p))]
    vp = vp + [sum(y * p)]
    
df = pandas.DataFrame({
    "variable": k_val,
    "rsquared": r2,
    "auc": roc_auc,
    "vrais_negatifs": vn,
    "faux_negatifs": fn,
    "faux_positifs": fp,
    "vrais_positifs": vp
})
df

Arbres de décision¶

Cette mthode itrative cherche, chaque noeud, la varaible explicative sparant le mieux les modalits prdire. A la racine de l'arbre, il y a donc tous les individus, et la population est spare en deux sous-population pour dans chaque fils. L'algorithme permet de dfinir quelle variable, et surtout quel seuil (dans le cas d'une variable quantitative) ou quelle rpartition de modalits (dans le cas d'une variable qualitative), permet de mieux sparer les classes prdire.

Cette mthode est intressante, car elle permet de voir rapidement quelles sont les variables ayant le plus d'impact sur la classe prdire (ce sont celles en premier dans l'arbre). Il existe diffrentes mthodes dans la faon de choisir la variable (CART par dfaut dans sklearn). Son dfaut principal est le sur-apprentissage possible, si l'arbre est trop profond. On parle alors d'lagage pour le rendre plus gnralisable.

Réseaux de neurones¶

On parle ici de perceptron (simple couche ou multi-couches, en fonction de la complexit que l'on souhaite). L'ide est de connecter des neurones d'entre (qui prendront les valeurs des variables explicatives) un neurone de sortie (ou plusieurs en fonction de la variable expliquer donc). Entre les deux, il y a une ou plusieurs couches de neurones cachs, chacun compltement connect aux neurones de la couche prcdente et de la couche suivante.

A partir de poids alatoires sur chaque liaison entre neurones, l'algorithme propage l'erreur entre la valeur observe et la valeur prdite par le rseau chaque couche, avec une mthode de back-propagation. Celle-ci permet de corriger les poids, afin que les neurones s'activent correctement. Cette mthode prend plusieurs paramtres comme le nombre de couches, la taille de chaque couche, la fonction d'activation d'un neurone, et d'autres encore.

Un des dfauts majeurs est la difficult, principalement dans le cadre d'un rseau multi-couches, percevoir l'impact d'une variable par rapport une autre. L'autre dfaut tant le nombre important de paramtres, et la variabilit des rsultats en fonction des valeurs de ceux-ci. Malgr cela, les performances sont plutt trs intressantes.

Ici, on dcide de n'avoir aucune couche cach, donc les neurones d'entres sont directement connects au neurone de sortie. Dans le cas d'une variable expliquer quantitative, cela revient au modle linaire gnralis, pour information.
